Linux Tutorial

발표자: 박대현

발표 일자: 2018년 7월 6일



DNN을 이용한 연구가 활발히 진행되고 있습니다. 현재 널리 사용하는 DNN 프레임워크의 대다수가 Python 프로그래밍 언어를 기반으로 하며, 기반 OS로 Linux - 특히 Ubuntu 및 그 Flavored derivatives - 를 채택하고 있습니다. Google의 Tensorflow, Facebook에서 공개한 Torch 및 PyTorch 등 다양한 Frameworks가 Linux 시스템에서 우선 개발된 뒤 Windows로 포팅되는 구조를 취하고 있으며, 속도 또한 tensorflow-gpu 기준 Linux에서 사용하는 것이 (별도의 코드 수정 없이 OS만 변경하더라도) 적게는 2배에서 크게는 10배 이상 빨라지는 것으로 보고되고 있습니다. 따라서 DNN을 이용하는데 있어서 Linux 사용은 필수적이라 할 것입니다. (물론 MS에서 개발하는 CNTK 라이브러리는 Windows 빌드를 기본으로 지원합니다.)


우리 연구실도 NVIDIA GeForce 1080Ti를 장착한 7대의 DNN 서버를 구축하였고, 그 외에도 개인에게 할당된 서버가 다수 있는 것으로 알고 있습니다. 이 서버들은 기반 OS로 Lubuntu 16.04LTS를 설치하여 가동중입니다. 많은 연구원들의 도움으로 서버 설치와 PyCharm을 이용한 Python 코드 원격 실행에 대한 매뉴얼 북은 어느 정도 완성된 것 같습니다. (https://github.com/YonseiMVP/linux_manual 등 연구실 GitHub 참고) 반면, 서버에서 DNN 코드를 돌리다보면 불가피하게 Linux 환경에서 일부 작업을 해야 하는 경우가 발생하게 됩니다. 간단하게는 파일 복사/이동/삭제부터, 소스코드를 직접 서버에서 수정하고 실행하거나, 나아가서는 시스템 서비스를 설정하고 구동해야 하는 경우도 발생하는 것이지요. 그러나 Windows 그래픽 환경에 익숙한 사용자에게 Linux, 그리고 콘솔 환경은 매우 낯선 것이 사실입니다.


본 튜토리얼의 대상은 Linux, 그리고 콘솔 환경의 사용 경험이 적은 사용자입니다. Linux의 뿌리는 UNIX이며, 이는 "프레임워크"라 부르는 대형 컴퓨터를 대상으로 개발되어, 복수의 사용자가 동시에 컴퓨팅 자원을 공유하면서 사용하는 것을 전제합니다. 이를 이어받은 Linux 또한 복수 사용자를 전제하므로, Windows와 작업 환경이 매우 다릅니다. 게다가 콘솔이라는, 검은 배경에 흰 글씨만 둥둥 떠다니는 환경에서 작업해야 합니다. 참 곤란하죠. 본 튜토리얼에서는 이렇게 처음 접하는 Linux 환경, 콘솔에서 직접 키보드를 두드려서 명령어를 입력하여 작업하는 방법을 연습할 것입니다. 물론 그 전에 원격 접속으로 "서버에 접속하는" 것부터 시작해야 하겠지요. 여러분이 직접 시스템 관리를 수행하는 것은 아니므로 내용을 방대하게 다루지는 않을 것입니다. 서버에 원격 접속하여, 인터넷 상의 파일을 직접 다운로드 받고, 압축을 풀고 파일과 디렉터리를 정리하며, 수십 줄 정도의 코드를 바로 작성 및 수정하여 실행할 수 있다면 본 튜토리얼을 성공적으로 마친 것이라 할 수 있습니다. 또한 이미 이 정도 작업이 가능하다면 굳이 이 튜토리얼에 참여할 필요도 없습니다.


이 튜토리얼은 단지 연구를 하는데 있어서 필요한 도구 사용법을 익히는 것이기 때문에, 연구자로서의 호기심을 그다지 자극하지도 않을 것이며(즉 재미가 없을 것이며) 연구실 세미나의 본래 목적과도 어긋납니다. 그럼에도 불구하고, 앞으로 Linux 환경에서 작업해야 하는 경우가 매우 많이 늘어날 것이므로 진행 필요성이 있어 진행하게 되었습니다. 시간이 되는 분들은 가급적 참여하여 주시기를 부탁드립니다. 튜토리얼을 진행함에 있어서 현장 실습을 진행하는게 유리하다고 보므로, 노트북을 지참하는걸 권장합니다. 서버에 접속해서 작업할 것이므로, Linux를 설치할 필요는 없습니다. 대신 PuTTY, 그리고 WinSCP 최신 버전은 설치하여 오시기 바랍니다.
